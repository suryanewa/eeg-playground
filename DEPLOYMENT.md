# Vercel Deployment

Brandy is a static site served from the repository root. Vercel builds `dist/app.js` and `dist/env.js` at deploy time and serves `index.html`, `styles.css`, and `assets/` from the project root.

## One-time Vercel setup

1. Import the Git repository into [Vercel](https://vercel.com/new).
2. Confirm project settings (also defined in `vercel.json`):
   - **Framework Preset:** Other
   - **Build Command:** `npm run build`
   - **Output Directory:** `.`
   - **Install Command:** `npm ci`
   - **Node.js Version:** 20.x
3. Add environment variables (Production and Preview):

```bash
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_PUBLISHABLE_KEY=your-publishable-or-anon-key
EEG_PROJECT_ID=00000000-0000-4000-8000-000000000001
```

Only use a Supabase publishable key or legacy anon key in Vercel. Never use a service role key.

4. Deploy.

`scripts/write-env.js` reads those variables during the build and writes `dist/env.js`, which the app loads before `dist/app.js`.

## Supabase

Run the SQL migration in:

```text
supabase/migrations/20260702184358_client_logo_voting.sql
```

Then create Supabase Auth users and add client users to `project_members`.

If auth is enabled, add your Vercel production and preview URLs to the Supabase project's allowed site URLs.

## What ships on deploy

- `index.html` and `styles.css`
- `assets/` including `og-image.png` for social previews
- `dist/app.js` generated by esbuild
- `dist/env.js` generated from Vercel environment variables

`dist/env.js` is gitignored locally and must be produced by the build step on Vercel.

## Local production check

```bash
npm ci
SUPABASE_URL=https://your-project-ref.supabase.co \
SUPABASE_PUBLISHABLE_KEY=your-publishable-or-anon-key \
EEG_PROJECT_ID=00000000-0000-4000-8000-000000000001 \
npm run build
npm run dev
```

Open `http://localhost:4173`.

## Regenerating the OG image

The share image is committed at `assets/og-image.png`. Regenerate locally (requires ImageMagick):

```bash
npm run og:image
```

## Routes

- `/` serves `index.html`
- `/admin` rewrites to `index.html` for the admin view
